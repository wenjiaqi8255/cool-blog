# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-30
**Phases:** 1-4
**Total Plans:** 17

## Overview

Visual impact meets content depth — Portfolio showcases work through striking visuals; Articles provide deep technical content with excellent readability.

## Phases

### Phase 1: Foundation & Bento Grid

**Goal**: Users see a visually striking responsive Bento Grid layout deployed to Cloudflare Pages with the core visual identity established.
**Depends on**: Nothing (first phase)
**Plans**: 5 plans

Plans:
- [x] 01-01: Project scaffolding with Astro, Cloudflare adapter, Tailwind CSS v4, and test infrastructure
- [x] 01-02: Card components (Image, Text, Terminal, Stats) with hover animations and GitHub API
- [x] 01-03: Configuration-driven Bento Grid with responsive breakpoints
- [x] 01-04: Navigation system with tabs, subscribe modal, and main pages
- [x] 01-05: Cloudflare Pages deployment configuration and documentation

**Details:**
- Bento Grid with 4-column layout, 4px gaps, 320px min card height
- Card variants: light (#F7F7F7) and dark (#111111)
- Typography: Inter (200-600 weights), JetBrains Mono for code
- Hover effects: grayscale-to-color on images, smooth transitions on cards

### Phase 2: Content System

**Goal**: Users can browse, read, and search technical articles with excellent readability and code syntax highlighting.
**Depends on**: Phase 1
**Plans**: 4 plans

Plans:
- [x] 02-01: Astro Content Collections with Zod schemas for articles
- [x] 02-02: Article list and individual article pages with layouts
- [x] 02-03: Syntax highlighting with Shiki
- [x] 02-04: Search and filter functionality (client-side)

**Details:**
- Markdown content from Git-managed files
- GitHub-dark Shiki theme for code blocks
- Client-side search by title, filter by tags
- Responsive article card layouts

### Phase 3: Newsletter & Backend

**Goal**: Users can subscribe to the newsletter and receive confirmation, with emails stored securely in Neon Postgres.
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:
- [x] 03-01: Neon Postgres database setup with Drizzle ORM
- [x] 03-02: Newsletter subscription form (React island) and API endpoints
- [x] 03-04: Confirmation and resend email functionality

**Details:**
- Neon Postgres with neon-http driver (faster for serverless)
- Drizzle ORM for type-safe database operations
- Resend API integration for email delivery
- Duplicate email prevention via UNIQUE constraint

### Phase 4: SEO & Launch

**Goal**: The site is discoverable by search engines and social platforms with proper metadata and feeds.
**Depends on**: Phase 3
**Plans**: 5 plans

Plans:
- [x] 04-01: Meta tags, Open Graph, and Twitter Cards
- [x] 04-02: RSS feed generation
- [x] 04-03: Sitemap and robots.txt
- [x] 04-04: Dynamic OG image generation
- [x] 04-05: Gap closure: Wire article OG images

**Details:**
- Open Graph images generated via SVG (compatible with Cloudflare prerender)
- Article-specific OG images with title and KERNEL_PANIC branding
- Sitemap with clean URLs, mobile-responsive design
- RSS feed with full article content and images

## Milestone Summary

**Key Decisions:**
- Used Tailwind CSS v4 without @astrojs/tailwind integration (CSS-first configuration)
- Used Astro 6 content.config.ts format with glob loader
- Used neon-http driver for faster serverless queries
- Used SVG instead of PNG for OG images (Cloudflare compatibility)

**Issues Resolved:**
- Peer dependency conflict with @astrojs/tailwind@6.0.2 vs Astro 6.1.1 → resolved with Tailwind v4 CSS-first approach
- ImageCard test file existed before component → removed to allow test infrastructure to work
- Route collision from duplicate articles.astro → removed duplicate

**Technical Debt:**
- No formal audit performed (project built with gsd-tools, not separately audited)
- No E2E tests written (infrastructure present but not used)
