<!-- From 11-CONTEXT.md: Multi-region routing architecture decision -->

Architecture:
```
Domestic users → Aliyun DNS (境内线路) → Aliyun CDN → Zeabur (Node.js server)
International users → Aliyun DNS (境外线路) → Cloudflare SaaS → Cloudflare Pages (Workers)
Default fallback → Aliyun DNS (默认线路) → Aliyun CDN → Zeabur
```

Key requirements from user decision:
1. Aliyun DNS 智能解析 with three lines (境内/境外/默认)
2. ALL CNAME records (no A records) to avoid Aliyun DNS cache pollution bug
3. Aliyun CDN for domestic with Zeabur origin
4. Cloudflare SaaS Custom Hostname for international
5. Verification via itdog.cn (domestic) and curl --resolve (international)

  <name>Task 1: Aliyun CDN Setup</name>
  <action>
Set up Aliyun CDN for domestic acceleration:

1. Log in to [Aliyun CDN Console](https://cdn.console.aliyun.com)

2. Click "添加加速域名" (Add Acceleration Domain)

3. Configure the加速域名:
   - **加速域名**: Your main domain (e.g., `kernel-panic.dev`)
   - **业务类型**: Choose "小文件" (Small Files) for blog content
   - **源站类型**: Choose "域名" (Domain)
   - **源站地址**: Your Zeabur project domain (e.g., `kernel-panic.zeabur.app`)

4. Configure HTTPS:
   - Go to "HTTPS配置" tab
   - Certificate Type: "免费DV证书" (Free DV Certificate)
   - Validation Method: "DNS自动验证" (DNS Automatic Validation)
   - Click "确定" to enable HTTPS

5. Note the CNAME target provided:
   - Copy the CNAME record: `yourdomain.com.w.alikunlun.com`
   - This will be used in Aliyun DNS configuration

6. Wait for certificate validation to complete
   - Aliyun will automatically add DNS validation records
   - Usually takes 5-10 minutes

**DO NOT:**
- Use "IP地址" as源站类型 (Zeabur domains can change)
- Use "自定义源站" unless you have specific requirements
- Skip HTTPS configuration (required for proper routing)
- Note: Use CNAME records only in DNS (no A records) to avoid Aliyun DNS cache bug
  </action>

<task type="checkpoint:human-action">
  <name>Task 2: Cloudflare SaaS Custom Hostname Setup</name>
  <action>
Set up Cloudflare SaaS for international routing:

1. **Register a domain with Cloudflare** (if not already):
   - This can be ANY domain (not necessarily your main blog domain)
   - Point the domain's NS records to Cloudflare
   - Example: Register or use a secondary domain like `yourdomain-cf.com`

2. Log in to [Cloudflare Dashboard](https://dash.cloudflare.com)

3. Navigate to the domain registered with Cloudflare

4. Go to "SSL/TLS" → "Custom Hostnames"

5. Click "Add Custom Hostname"

6. Configure custom hostname:
   - **Hostname**: Your main blog domain (e.g., `kernel-panic.dev`)
   - **SSL**: Select "Flexible SSL" (easiest setup) or "Full SSL" (if origin supports it)
   - **Fallback Origin**: Enter your Zeabur or Cloudflare Pages address
     - For Zeabur: `https://kernel-panic.zeabur.app`
     - For Cloudflare Pages: Use the Pages URL
   - Click "Add"

7. **Add TXT verification record**:
   - Cloudflare will provide a TXT record for verification
   - Format: `_cf-custom-hostname.yourdomain.com TXT verification-code`
   - Copy this record

8. **Add the TXT record to Aliyun DNS**:
   - Log in to Aliyun DNS console
   - Add TXT record with the verification code from Cloudflare
   - Wait for DNS propagation (may take up to 24 hours)

9. Note the SaaS hostname target:
   - Cloudflare SaaS provides a hostname like: `xxx.yourdomain-cf-sas.com`
   - This will be used in Aliyun DNS境外线路

**DO NOT:**
- Try to add a custom hostname to a domain not hosted on Cloudflare
- Skip the TXT verification step (hostname won't work without it)
- Use "Full (Strict)" SSL unless your origin has valid SSL certificate
  </action>

<task type="checkpoint:human-action">
  <name>Task 3: Aliyun DNS Intelligent Routing Configuration</name>
  <action>
Configure Aliyun DNS intelligent routing with three lines (境内/境外/默认):

1. Log in to [Aliyun DNS Console](https://dns.console.aliyun.com)

2. Select your domain

3. Delete existing A/CNAME records for the main domain (@)
   - Click "删除" on any existing @ records

4. Add new CNAME records with intelligent routing:
   - Click "添加记录"
   - **记录类型**: CNAME
   - **主机记录**: @ (for main domain) or www (for www subdomain)
   - **记录值**: See routing table below
   - **TTL**: 600 seconds (default)

5. Configure three lines (境内/境外/默认) in a single CNAME record:

| 线路类型 | 记录值 | Target |
|----------|--------|--------|
| 境内 | `yourdomain.com.w.alikunlun.com` | Aliyun CDN → Zeabur |
| 境外 | `xxx.yourdomain-cf-sas.com` | Cloudflare SaaS → Cloudflare Pages |
| 默认 | `yourdomain.com.w.alikunlun.com` | Aliyun CDN → Zeabur (fallback) |

6. **CRITICAL**: ALL THREE lines must be CNAME records
   - Do NOT use A records (causes Aliyun DNS cache pollution bug)
   - Each line should have the same CNAME type with different targets

7. Add the TXT verification record for Cloudflare SaaS (if not already added):
   - **记录类型**: TXT
   - **主机记录**: `_cf-custom-hostname`
   - **记录值**: The verification code from Cloudflare SaaS setup
   - **线路**: 默认

8. Wait for DNS propagation
   - Usually takes 10-30 minutes for changes to take effect
   - Some regions may take up to 24 hours

**DO NOT:**
- Mix A records with CNAME records (causes cache pollution bug)
- Use "境外_北美" or specific regional lines (use generic "境外")
- Skip the "默认" line (needed as fallback)
- Forget the TXT verification record for Cloudflare SaaS
  </action>

   ## Verification Procedures

   ### Test Domestic Routing

   Use [itdog.cn](https://www.itdog.cn/) for comprehensive domestic multi-node testing:

   1. Visit https://www.itdog.cn/
   2. Enter your domain
   3. Select "HTTP" or "HTTPS" detection
   4. Click "立即检测"

      - CDN nodes detected: Aliyun CDN (Tengine server header)
      - No routing to international endpoints

   **Alternative: curl command**
   ```bash
   curl -v https://yourdomain.com 2>&1 | grep -E "Server:|Age:|CF-Cache-Status"
   ```
   Expected output for domestic routing:
   - `Server: Tengine` (Aliyun CDN)
   - `Age:` header present (CDN cached response)
   - No `CF-Cache-Status` header

   ### Test International Routing

   Test from an international location or use a VPN:

   ```bash
   curl -v https://yourdomain.com 2>&1 | grep -E "Server:|CF-Ray:|CF-Cache-Status"
   ```
   Expected output for international routing:
   - `Server: cloudflare` or no server header (Cloudflare)
   - `CF-Ray:` header present with airport code
   - `CF-Cache-Status:` header present

   **Direct IP test** (bypasses DNS):
   ```bash
   # Get Cloudflare IP for your SaaS hostname
   dig +short xxx.yourdomain-cf-sas.com

   # Test direct connection to Cloudflare
   curl -v --resolve yourdomain.com:443:[cloudflare-ip] https://yourdomain.com
   ```

   ### Verify DNS Records

   ```bash
   # Check domestic DNS resolution (from China)
   dig yourdomain.com

   # Check international DNS resolution
   dig @8.8.8.8 yourdomain.com

   # Verify CNAME chain
   dig +trace yourdomain.com
   ```

   ### Test Both Platforms Directly

   ```bash
   # Test Zeabur directly
   curl -v https://your-zeabur-domain.zeabur.app

   # Test Cloudflare Pages directly
   curl -v https://your-.pages.dev
   ```

   ### Common Verification Issues

   **Both routes go to same platform**:
   - Check Aliyun DNS configuration (all three lines present?)
   - Verify CNAME targets are correct
   - Check DNS propagation (wait 24 hours)

   **Domestic routing shows Cloudflare**:
   - 境内 line might be pointing to wrong target
   - Check Aliyun CDN is configured correctly

   **International routing shows Aliyun CDN**:
   - 境外 line might be pointing to wrong target
   - Check Cloudflare SaaS custom hostname is active

   **SSL certificate errors**:
   - Verify HTTPS is enabled on Aliyun CDN
   - Verify custom hostname SSL is active on Cloudflare SaaS
   - Check TXT verification record is present
   ```
