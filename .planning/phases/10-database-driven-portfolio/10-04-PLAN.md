# Plan 10-04: Visitor Stats Integration

**Goal:** Replace "Weekly Commits" with visitor count display

**Duration:** 1 hour

**Dependencies:** Plan 10-03 (BentoGrid integration)

---

## Context

Current mockup has a "Weekly Commits" stats card. We need to replace it with visitor count to make it more relevant for a personal blog/portfolio.

## Requirements

### Functional
- Display visitor count in stats card
- Real-time or near-real-time data
- Fallback when data unavailable
- Match Bento aesthetic

### Technical
- Integrate with Vercel Analytics or simple counter
- Cache data to avoid rate limits
- Handle errors gracefully
- Server-side rendering compatible

## Implementation Options

### Option A: Vercel Analytics API (Recommended)

Pros:
- Built into Vercel deployment
- Accurate data
- No additional services

Cons:
- Requires Vercel Pro plan for API access
- Rate limiting

### Option B: Simple Counter (Upstash Redis)

Pros:
- Simple implementation
- No external API dependencies
- Fast

Cons:
- Less accurate (doesn't track unique visitors well)
- Requires Upstash Redis setup

### Option C: Third-party Service (Plausible, Umami)

Pros:
- Privacy-focused
- Detailed analytics

Cons:
- Additional service to manage
- May require self-hosting

## Recommended Implementation: Option B (Simple Counter)

For Phase 10, use a simple counter approach. Can migrate to Vercel Analytics later if needed.

### Step 1: Create Visitor Counter

File: `src/lib/visitor-counter.ts`

```typescript
import { Redis } from '@upstash/redis';

// Initialize Redis client (if using Upstash)
const redis = new Redis({
  url: import.meta.env.UPSTASH_REDIS_REST_URL,
  token: import.meta.env.UPSTASH_REDIS_REST_TOKEN
});

const COUNTER_KEY = 'visitor_count';
const CACHE_TTL = 60; // Cache for 60 seconds

/**
 * Increment visitor count
 * Call this on each page load
 */
export async function incrementVisitorCount(): Promise<number> {
  try {
    const count = await redis.incr(COUNTER_KEY);
    return count;
  } catch (error) {
    console.error('Failed to increment visitor count:', error);
    return 0;
  }
}

/**
 * Get current visitor count
 * Cached to avoid hitting Redis on every request
 */
export async function getVisitorCount(): Promise<number> {
  try {
    const count = await redis.get<number>(COUNTER_KEY);
    return count ?? 0;
  } catch (error) {
    console.error('Failed to get visitor count:', error);
    return 0;
  }
}

/**
 * Get visitor count with caching
 * Returns cached value if available, otherwise fetches from Redis
 */
let cachedCount: number | null = null;
let cacheExpiry: number = 0;

export async function getVisitorCountCached(): Promise<number> {
  const now = Date.now();

  if (cachedCount !== null && now < cacheExpiry) {
    return cachedCount;
  }

  const count = await getVisitorCount();
  cachedCount = count;
  cacheExpiry = now + CACHE_TTL * 1000;

  return count;
}
```

### Step 3: Increment Counter on Visit

Add client-side script to increment counter:

```typescript
---
// In index.astro
---

<script>
  // Track visitor (only on first visit per session)
  if (!sessionStorage.getItem('visited')) {
    fetch('/api/track-visit', { method: 'POST' })
      .then(() => sessionStorage.setItem('visited', 'true'))
      .catch(err => console.error('Failed to track visit:', err));
  }
</script>
```

### Step 4: Create Tracking API

File: `src/pages/api/track-visit.ts`

```typescript
import type { APIRoute } from 'astro';
import { incrementVisitorCount } from '../../lib/visitor-counter';

export const POST: APIRoute = async () => {
  try {
    await incrementVisitorCount();
    return new Response(JSON.stringify({ success: true }), {
      status: 200,
      headers: { 'Content-Type': 'application/json' }
    });
  } catch (error) {
    console.error('Failed to track visit:', error);
    return new Response(JSON.stringify({ success: false }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' }
    });
  }
};
```

## Testing

1. **Basic Functionality:**
   - Visit homepage → counter increments
   - Refresh page → counter does NOT increment (session check)
   - Close browser, revisit → counter increments

2. **Edge Cases:**
   - Redis unavailable → shows "---"
   - Rate limiting → cached value used
   - First visit ever → shows "1"

3. **Performance:**
   - Page load time not affected
   - API call happens asynchronously
   - Cached value served instantly

## Alternative: Vercel Analytics Integration

If you have Vercel Pro plan, use Vercel Analytics API instead:

```typescript
import { Analytics } from '@vercel/analytics';

export async function getVisitorCount(): Promise<number> {
  try {
    const analytics = new Analytics({
      token: import.meta.env.VERCEL_ANALYTICS_TOKEN
    });

    const response = await analytics.get({
      endpoint: '/v1/analytics',
      query: {
        period: '30d',
        metric: 'pageviews'
      }
    });

    return response.data.pageviews;
  } catch (error) {
    console.error('Failed to get visitor count:', error);
    return 0;
  }
}
```
