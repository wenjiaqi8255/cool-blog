# Plan 10-01: Schema Extension & Migration

**Goal:** Add image field to articles table for portfolio card display

**Duration:** 30 minutes

**Dependencies:** Phase 8 complete (Drizzle ORM setup)

---

## Context

Current articles table has no image field. For portfolio cards (especially featured/image variants), we need to associate images with articles.

## Requirements

### Functional
- Add optional `image` field to articles table
- Field stores URL to image (external or internal)
- Migration should not affect existing data
- Schema should be backwards compatible

### Technical
- Use Drizzle ORM migration system
- Field type: `text` (stores URL string)
- Optional field (NULL allowed)
- No default value (NULL by default)

## Implementation

### Step 1: Update Drizzle Schema

File: `src/db/schema.ts`

```typescript
export const articles = pgTable('articles', {
  id: serial('id').primaryKey(),
  title: text('title').notNull(),
  slug: text('slug').notNull().unique(),
  date: timestamp('date').notNull(),
  tags: text('tags').array(),
  excerpt: text('excerpt'),
  body: text('body').notNull(),
  image: text('image'), // NEW: Optional image URL
  status: text('status').notNull().default(ArticleStatus.DRAFT),
  deleted_at: timestamp('deleted_at'),
  created_at: timestamp('created_at').defaultNow().notNull(),
  updated_at: timestamp('updated_at').defaultNow().notNull(),
});
```

### Step 2: Generate Migration

```bash
npm run db:generate
```

This creates a migration file in `drizzle/` directory.

### Step 3: Review Migration SQL

Expected SQL:
```sql
ALTER TABLE articles ADD COLUMN image text;
```

### Step 4: Apply Migration

```bash
npm run db:migrate
```

### Step 5: Update TypeScript Types

The Article type in `src/lib/articles.ts` will automatically include the new field:

```typescript
export interface Article {
  // ... existing fields
  image: string | null; // NEW
}
```

### Step 6: Test Migration

1. Check existing articles still load
2. Verify new field is accessible
3. Test NULL value handling

## Acceptance Criteria

- [ ] Migration generated successfully
- [ ] Migration applied without errors
- [ ] Existing articles unaffected
- [ ] New field accessible in TypeScript
- [ ] NULL values handled correctly
- [ ] Can update article with image URL via MCP tools

## Rollback Plan

If migration fails:
```sql
ALTER TABLE articles DROP COLUMN image;
```

## Notes

- Image URLs can be external (Unsplash, etc.) or internal paths
- No validation at database level (validation happens in application layer)
- Future enhancement: Add image upload to Vercel Blob
